# 交互规范

## 一、目标

本文件用于约束 Mango 的关键交互，避免后续不同版本出现相互冲突的操作逻辑。

## 二、关键交互对象

- 任务输入
- 计划生成
- 权限批准
- 执行时间线
- 任务回顾
- 工作区切换

## 三、任务输入交互

### 目标

帮助用户清楚表达“要做什么”，而不是让用户输入大量底层命令。

### 要求

- 文案鼓励输入目标而不是命令
- 输入框允许中长文本
- 必须能看出当前任务属于哪个工作区

## 四、计划生成交互

### 目标

让用户先理解“准备怎么做”，再决定是否执行。

### 要求

- 计划必须是结构化的
- 至少包括：标题、摘要、步骤、权限请求
- 用户不应在未看到计划时直接进入执行

## 五、权限批准交互

### 目标

让用户能判断“要不要给这个权限”。

### 要求

- 必须展示权限类型
- 必须展示权限原因
- 必须区分风险等级
- 必须有明确批准动作

## 六、执行中交互

### 目标

让用户知道任务是否还活着、正在做什么、是否出现问题。

### 要求

- 有明确运行状态
- 有事件时间线
- 长任务不可表现为静默空白
- 失败事件必须明显可见

## 七、任务完成交互

### 目标

让用户快速回答“这次任务完成了吗，是否值得继续”。

### 要求

- 必须有摘要
- 必须有变更文件或关键证据
- 必须能判断结果状态

## 八、工作区切换交互

### 目标

降低“我现在到底在哪个项目上”的认知负担。

### 要求

- 当前工作区高亮
- 工作区至少显示路径、分支、状态摘要
- 切换工作区后主任务上下文要同步刷新
