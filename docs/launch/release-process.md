# 发布流程

## 一、目标

本文件定义 Mango 的版本发布方式，确保后续从内部迭代走向公开测试时，流程一致、职责清晰。

更细的版本号、发布通道、Tag 和安装包命名规则，统一参见 `versioning-and-release-standards.md`。

## 二、版本通道

### `nightly`

- 面向内部快速验证
- 允许存在已知问题
- 用于观察新功能的基本稳定性

### `beta`

- 面向小规模外部试用
- 要求主流程稳定
- 允许存在非阻塞缺陷

### `stable`

- 面向更广泛用户
- 需要更严格的回归验证

## 三、发布前检查

发布前必须完成：

- 测试通过
- 类型检查通过
- 构建通过
- 变更说明完成
- 已知问题确认
- 文档同步完成

## 四、发布产物

每次发布应至少包含：

- 版本号
- 安装包
- 变更说明
- 已知问题
- 下载方式

## 五、发布后动作

- 检查下载链接
- 检查安装与启动
- 收集首批问题反馈
- 记录是否需要热修复

## 六、版本节奏建议

- 日常开发：`nightly`
- 里程碑打磨：`beta`
- 公开传播：`stable`
