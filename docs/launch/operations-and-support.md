# 运维与支持

## 一、目标

这份文档用于定义 Mango 在进入对外试用后，如何处理问题、反馈、紧急修复和用户支持。

## 二、问题来源

- GitHub Bug Report
- GitHub Feature Request
- 应用内反馈入口
- 社区评论与试用反馈

## 三、问题分级建议

### P0

- 应用无法启动
- 主流程完全不可用
- 严重权限或安全问题

### P1

- 关键功能高频失败
- 任务结果严重不可信
- 安装或更新大量失败

### P2

- 有替代路径的缺陷
- 一般体验问题

### P3

- 视觉细节、优化建议、低优先级增强

## 四、事故处理流程

1. 收集问题信息
2. 判断影响范围
3. 判断问题等级
4. 给出临时规避方案
5. 排期修复
6. 补充回归测试与文档

## 五、支持原则

- 先保证主流程稳定
- 先解决影响面最广的问题
- 修复后必须补验证与文档
