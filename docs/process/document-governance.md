# 文档治理规则

## 一、为什么需要治理规则

文档如果没有治理规则，过几周就会变成一堆“看起来很多，但没人敢信”的文件。Mango 要避免这种情况。

## 二、文档分级

### 一级文档：方向性文档

- 愿景与定位
- PRD
- 路线图

作用：决定做什么、不做什么。

### 二级文档：实施性文档

- 技术架构
- 模块契约
- 测试策略
- 发布流程
- 工程规范文档族

作用：决定怎么做、怎么验。

### 三级文档：操作性文档

- 清单
- 风险记录
- 贡献指南

作用：保证执行一致性。

## 三、更新原则

- 改方向，先改一级文档
- 改方案，先改二级文档
- 改流程，先改三级文档

## 四、失效判定

满足以下任一条件时，文档应视为失效并优先修复：

- 与代码行为明显冲突
- 与当前里程碑目标冲突
- 新成员阅读后无法指导工作

## 五、维护要求

- 每个里程碑至少巡检一次文档
- 大改动 PR 必须说明修改了哪些文档
- 不允许长时间存在“文档以后再补”
