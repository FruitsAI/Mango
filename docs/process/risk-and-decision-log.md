# 风险与决策记录

## 一、使用方式

本文件用于记录 Mango 当前阶段的重要风险和关键决策，避免团队反复讨论同一个问题。

## 二、当前关键决策

### 决策 1：v1 先做开发者桌面 Agent 工作台

- 原因：用户更集中、反馈更直接、验证路径更短
- 影响：暂不面向通用办公人群扩展

### 决策 2：技术路线选择 Electron + React

- 原因：跨平台桌面能力成熟、生态丰富、工程推进快
- 影响：先接受包体和 Electron 生态的典型成本

### 决策 3：先做 mock adapter，再接真实 adapter

- 原因：先把闭环体验和状态边界做稳
- 影响：当前产品价值验证还不完整，后续必须尽快接入真实执行

### 决策 4：持久化先用 JSON

- 原因：搭骨架更快
- 影响：后续必须规划 SQLite 升级

## 三、当前主要风险

### 风险 1：真实 adapter 接入晚于产品预期

- 影响：会导致产品长期停留在演示级状态
- 建议：下一阶段优先处理

### 风险 2：权限模型只停留在展示层

- 影响：会削弱 Mango 的核心差异化
- 建议：尽快把权限逻辑接入真实执行闸门

### 风险 3：文档维护跟不上实现

- 影响：团队会重新回到口头共识和临时判断
- 建议：把文档更新纳入完成标准
