# 验收标准

## 一、目标

本文件定义 Mango 当前阶段的“验收标准”，用于避免不同角色对“是否完成”理解不一致。

## 二、产品层验收

### 主流程验收

- 用户可以输入任务目标
- 用户可以看到结构化计划
- 用户可以看到权限请求
- 用户可以启动执行
- 用户可以看到执行事件
- 用户可以看到结果摘要

### 理解性验收

- 用户能知道当前工作区
- 用户能知道当前任务状态
- 用户能知道当前使用哪个 adapter
- 用户能知道本次执行请求了哪些权限

## 三、工程层验收

- `npm run test` 通过
- `npm run typecheck` 通过
- `npm run build` 通过

## 四、文档层验收

- 对应需求文档存在
- 对应技术设计文档存在
- 对应测试策略已更新
- 发布影响已同步到发布文档

## 五、不算完成的情况

- 代码可运行，但没有更新文档
- 本地能跑，但没有通过验证命令
- 交互做出来了，但用户无法理解当前状态
- 功能存在，但无法回答“为什么现在做这个”
