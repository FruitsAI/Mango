# 存储与安全设计

## 一、当前存储策略

当前 Mango 已切换为“`SQLite 主存储 + JSON 导入兜底`”策略：

- `desktop-state.db` 作为桌面端正式本地存储
- `desktop-state.json` 仅作为早期版本遗留状态的导入来源

当前 SQLite 负责保存：

- 当前活跃任务
- 历史任务
- 选中的工作区
- 桌面设置
- 执行事件
- schemaVersion 与 migration 版本

## 二、当前落地方式

当前实现采用两段式演进：

1. 早期阶段先用 JSON 快速验证持久化模型
2. 当前阶段切到 SQLite，并在首次读取空数据库时自动尝试导入旧 JSON 状态

这意味着：

- 新安装或新状态默认写入 SQLite
- 旧用户如果本地还保留 JSON 状态，会在第一次启动时自动迁入 SQLite
- JSON 不再作为长期正式主存储

## 三、为什么升级到 SQLite

随着 Mango 增长，JSON 会遇到这些问题，而 SQLite 更适合正式桌面产品：

- 支持任务、事件、工作区的结构化查询
- 支持 migration 与 schemaVersion 管理
- 支持后续恢复、搜索、索引与统计能力
- 支持更清晰的失败恢复与兼容升级策略

## 四、当前 SQLite 基线

当前桌面端已经建立：

- 首个 migration 基线
- migration catalog
- `schemaVersion`
- 旧 JSON 自动导入逻辑

当前仍未完成的部分：

- 全量查询优化
- 更细的索引策略
- 更完整的失败回滚与损坏修复
- 从 SQLite 进一步衍生审计与恢复工具

## 五、安全设计原则

### 原则 1：高风险能力默认受控

Shell、文件、网络、浏览器必须纳入权限模型。

### 原则 2：敏感内容不应默认上报

以下内容默认不应进入遥测：

- 原始提示词全文
- 文件内容
- 差异内容
- 环境变量
- 密钥

### 原则 3：执行必须留痕

为了可回顾和排错，Mango 至少要保留：

- 计划摘要
- 权限请求
- 执行事件
- 结果摘要

## 六、当前已定义的权限能力

- `shell`
- `filesystem`
- `network`
- `browser`

## 七、后续需要补齐的安全能力

- 工作区级授权范围
- 敏感目录保护
- 明确的删除类操作防护
- 更细的网络权限说明
- 更完整的审计记录

## 八、变更原则

只要本地存储结构或安全策略发生变化，必须同步更新：

- 本文档
- 测试
- 对应代码注释或契约类型
- 上线检查清单
